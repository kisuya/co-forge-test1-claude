# Ideation: StockWhy (주가 변동 원인 분석 서비스)

## Core Idea

보유 또는 관심 종목의 주가가 크게 변동했을 때, **왜 변동했는지**를 AI가 깊이 있게 분석해주는 독립 웹 서비스.

- **대상 시장**: 한국 + 미국 주요 기업
- **핵심 차별점**: 단순 뉴스 요약이 아닌, 유사 과거 케이스 비교 및 기업의 장기적 변화 맥락까지 포함한 심층 분석
- **사용자**: 1년 이상 투자 경험이 있는 일반 개인투자자
- **형태**: 웹 기반 대시보드 (증권사 계좌 불필요)

### 작동 방식

1. 사용자가 관심 종목(한국/미국) 등록
2. 주가 급변동 감지 시 AI가 뉴스, 공시, 시장 데이터를 수집
3. LLM 기반으로 변동 원인을 분석하고 유사 과거 사례와 장기 트렌드를 포함한 리포트 생성
4. 웹 대시보드에서 알림 및 리포트 확인

### Why Now

- 한국 개인투자자 1,423만 명 (2024년 기준), 투자 관심 지속적으로 높음
- LLM 기술 성숙 및 비용 하락으로 심층 분석 자동화가 현실적
- 증권사 AI 서비스는 자사 고객 종속 → 독립 서비스 수요 존재

## Technical Feasibility

**점수: 4/5 (Strong)**

### 데이터 소스 — 충분히 확보 가능

| 데이터 | 한국 | 미국 |
|--------|------|------|
| 주가/시세 | KRX Open API, PyKRX | Alpha Vantage, Finnhub |
| 공시/재무 | DART OpenAPI | SEC EDGAR, FMP API |
| 뉴스 | 네이버 뉴스, 뉴스핌 | Finnhub News, EODHD |

### AI 분석 — 기술적으로 가능

- LLM (GPT-4o, Claude 등)을 활용한 뉴스/공시 분석은 이미 검증된 접근
- 실시간이 아닌 "깊이 있는 분석"이라 배치 처리 가능 → 인프라 비용 절감
- 유사 케이스 매칭은 벡터 DB (예: Pinecone, Weaviate)로 구현 가능

### 복잡도

- **MVP**: 1-2개월 (한국 주요 종목 50개 + 미국 주요 종목 50개)
- **핵심 난이도**: 한국 뉴스 크롤링 품질, 유사 케이스 DB 구축
- **알려진 어려운 문제**: 변동 원인의 정확한 귀인(attribution) — 여러 요인이 동시에 작용할 때 어떤 것이 주요인인지 판단

## Market Opportunity

- 한국 개인투자자 1,423만 명 중 경험 있는 투자자 (1년 이상) 약 50% 추정 → ~700만 명
- 이 중 "왜 변동했는지" 능동적으로 알고 싶어하는 사용자 10% 추정 → ~70만 명
- 독립 유료 서비스 전환율 2% 추정 → ~14,000명
- 월 구독 9,900원 기준 → 연 매출 약 16.6억 원 (SOM)

글로벌 확장 시 미국 시장까지 포함하면 SOM 3-5배 확장 가능.

## Risks (Top 3)

### 1. 증권사 AI 기능 강화 (경쟁 리스크)
- **위험**: 토스, 키움, 한투 등이 자체 AI 분석을 심층화하면 독립 서비스 입지 약화
- **완화**: "증권사 어디든 상관없이 쓸 수 있는 포트폴리오 통합 분석"으로 포지셔닝. 분석 깊이에서 압도적 차이 유지

### 2. 유료 전환 장벽 (수익화 리스크)
- **위험**: 증권사 무료 기능이 있는데 별도 비용을 낼 유인 부족
- **완화**: 프리미엄 가치 명확히 전달 (무료 = 요약 1줄, 유료 = 유사 케이스 + 장기 분석 리포트). 프리미엄 모델로 시작

### 3. 변동 원인 귀인 정확도 (기술 리스크)
- **위험**: AI가 잘못된 원인을 제시하면 신뢰도 하락
- **완화**: 확신도(confidence) 표시, 복수 원인 병렬 제시, 출처 링크 제공으로 투명성 확보

## Recommendation: Conditional Go

**진행 가능하나, 다음 조건 충족 필요:**

1. **MVP에서 분석 깊이 차별화를 명확히 검증** — 토스 AI 시그널 대비 사용자가 체감하는 가치 차이가 있는지 20-30명 대상 테스트
2. **수익화 모델 초기 검증** — 프리미엄 분석에 대한 지불 의향 확인 (사전 등록/설문)
3. **핵심 기술 프로토타입** — 유사 케이스 매칭 + 장기 트렌드 분석의 품질이 "읽을 만한" 수준인지 확인

위 3가지 중 하나라도 부정적이면 피봇 또는 중단 검토.
